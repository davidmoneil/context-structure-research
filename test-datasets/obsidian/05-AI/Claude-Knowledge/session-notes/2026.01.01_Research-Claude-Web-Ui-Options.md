# Research: Claude Code Web UI Options

**Date**: 2026-01-01
**Status**: Research Complete - No Action Taken
**Decision**: Keep using CLI; add OpenWebUI Anthropic pipe for family if needed

## Context

Evaluated options for building a web interface on top of Claude Code to potentially:
- Provide session management UI
- Enable family access to Claude
- Integrate with n8n workflows

## Projects Reviewed

### CUI (github.com/wbopan/cui)

**Stats**: 1,052 stars | TypeScript | 488 tests | Apache-2.0

**Approach**: CLI spawning with JSONL stream parsing

**Pros**:
- Preserves hooks system (spawns actual CLI)
- Multi-model routing via @musistudio/llms
- Custom MCP permission server
- Good test coverage

**Cons**:
- MCP polling loop (1 req/sec indefinitely) - resource waste
- Environment variable filtering issues
- In-memory rate limiting (no persistence)

**Grade**: B+

### ClaudeCodeUI (github.com/siteboon/claudecodeui)

**Stats**: 5,230 stars | JavaScript | 0 tests | GPL-3.0

**Approach**: Direct Agent SDK integration (bypasses CLI)

**Pros**:
- Multi-provider support (Claude, Cursor, Codex)
- File explorer, Git integration, shell terminal
- TaskMaster AI project management
- Active development

**Cons**:
- Bypasses hooks system entirely (SDK approach)
- JWT tokens never expire (security issue)
- Hardcoded fallback secret
- 40k line monolithic App.jsx
- No tests

**Grade**: B+

**Neither project suitable** - CUI has resource issues, ClaudeCodeUI bypasses our hooks infrastructure.

## Development Path Analysis

### Option 1: Custom Build

**LOE**: 25-32 days

**Architecture**:
```
Browser (React) <--WebSocket--> Express Server <--spawn--> Claude CLI
                                      |                        |
                                      v                        v
                               Memory MCP              .claude/hooks/
```

**Phases**:
1. Core infrastructure (8-10 days) - CLI spawning, WebSocket, streaming
2. Essential features (10-12 days) - File browser, session management, MCP
3. Polish (7-10 days) - Mobile, notifications, dark mode

**Pros**: Full control, hooks preserved, complete feature parity
**Cons**: Significant LOE, ongoing maintenance burden

### Option 2: n8n Integration

**LOE**: 13 days

**Architecture**:
```
React Frontend <--WebSocket--> n8n Chat Trigger <--HTTP--> Claude Wrapper Service
                                                                   |
                                                                   v
                                                              Claude CLI
```

**Key limitation**: n8n Execute Command node doesn't support streaming. Requires wrapper service anyway.

**Pros**: Leverages existing n8n infrastructure
**Cons**: Still needs custom wrapper, more complex architecture

### Option 3: OpenWebUI Integration

**Option A - Anthropic Pipe (0.5-5.5 days)**:
- Adds Claude API to OpenWebUI
- NOT Claude Code - just chat
- Simple Python pipe function
- No hooks, no MCP, no file editing

**Option B - Full Proxy (11 days)**:
- WebSocket proxy spawning Claude CLI
- Full Claude Code features
- Complex integration

## Conclusions

### For Personal Use

**Recommendation**: Keep using CLI

Current CLI workflow is already well-optimized:
- Hooks working (audit-logger, session-tracker, docker-health-check)
- 7 MCP servers integrated
- 19+ slash commands built
- Session management via --resume flag

Web UI would provide marginal convenience improvements, not new capabilities.

### For Family Use

**Recommendation**: OpenWebUI Anthropic Pipe (0.5 days)

Family members don't need Claude *Code* - they need Claude *chat*:
- Simple questions and conversations
- Writing assistance
- No file editing or bash commands needed

The minimal pipe gives them Claude in a familiar interface without maintenance burden.

### For n8n Automation

**Recommendation**: Wait for concrete use cases

The wrapper service complexity only makes sense with specific automation needs. Currently speculative.

## Quick Win Option

If family access is needed later, implement OpenWebUI Anthropic pipe:

1. Create Python pipe function in OpenWebUI admin
2. Configure ANTHROPIC_API_KEY
3. Family gets Claude chat in familiar UI
4. No maintenance of custom service

**Reference**: OpenWebUI pipes documentation
