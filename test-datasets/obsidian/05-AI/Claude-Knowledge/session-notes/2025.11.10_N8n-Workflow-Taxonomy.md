# n8n Workflow Taxonomy & Naming Convention

**Created**: 2025-11-10
**Purpose**: Standardize n8n workflow naming and tagging for better organization and discoverability

## Naming Convention

### Format
```
[Category]: [Action/Purpose] [Details/Technology]
```

### Examples
- `Tool: Extract Google Drive Files`
- `AI Agent: Gmail Assistant (Llama 3.2)`
- `Monitor: Security Now Podcast → AI Summary`
- `Benchmark: Chain-of-Thought vs Basic Prompting`
- `Meta: AI-Powered Workflow Naming Assistant`

### Category Prefixes

| Prefix | Usage | Example |
|--------|-------|---------|
| **Tool:** | Reusable sub-workflows called by other workflows | `Tool: Check Multi-Calendar Availability` |
| **AI Agent:** | Conversational AI agents with chat interfaces | `AI Agent: Chat with Postgres Memory` |
| **Monitor:** | Scheduled workflows that watch external sources | `Monitor: Security Now Podcast → Email` |
| **Benchmark:** | Performance testing and comparison workflows | `Benchmark: Chain-of-Thought vs Basic` |
| **Meta:** | Workflows that manage n8n itself | `Meta: AI-Powered Workflow Naming` |
| **Automation:** | Standard automation flows (non-AI) | `Automation: Archive Old Gmail` |
| **Template:** | Reference implementations for reuse | `Template: OpenSearch Vector Embeddings` |
| **Training:** | Educational/learning workflows | `Training: Cannon's Test Workflow` |

### Naming Best Practices

1. **Be Specific**: Use action verbs (Extract, Monitor, Archive, Check)
2. **Include Technology**: Add model names, services in parentheses
3. **Show Flow**: Use arrows (→) to show data flow
4. **Avoid Typos**: Run spell check before finalizing
5. **No Versions**: Use workflow versioning feature or tags instead
6. **Remove Redundancy**: Don't repeat "n8n" or "workflow"
7. **Length Limit**: Keep under 60 characters if possible

## Tag Taxonomy

### 1. Status Tags (Mutually Exclusive)
Track workflow lifecycle state

| Tag | Description | Usage |
|-----|-------------|-------|
| `Production` | Live, actively used workflows | Replace "active: true" indicator |
| `DRAFT` | In development, not ready | Work in progress |
| `Template` | Reference implementation | For cloning/learning |
| `Training` | Educational purposes | Tutorials, demos |
| `Archive` | No longer used but kept | Historical reference |
| `Deprecated` | Being phased out | Migration in progress |

### 2. Type Tags (Mutually Exclusive)
Primary workflow function

| Tag | Description | Example Workflows |
|-----|-------------|-------------------|
| `Tool` | Reusable sub-workflow | File extractors, calendar checkers |
| `AI-Agent` | Conversational AI | Chat agents, assistants |
| `Automation` | Standard automation | Email processing, file sync |
| `Monitor` | Watches external sources | RSS, webhooks, scheduled checks |
| `Benchmark` | Performance testing | LLM comparisons, speed tests |
| `Meta` | Manages n8n itself | Workflow analyzers, health checks |
| `Integration` | Connects two systems | Sync between services |
| `Processing` | Data transformation | ETL, parsing, enrichment |

## Tag Application Strategy

### Minimum Required Tags
Every workflow should have at least:
1. **One Status Tag** (Production, DRAFT, Template, etc.)
2. **One Type Tag** (Tool, AI-Agent, Automation, etc.)

### Recommended Tags
Add these when applicable:
- Integration tags for each external service
- Technology tags for key capabilities
- Domain tag for business context

### Example Tag Sets

**Gmail Archival Workflow:**
- Status: `Production`
- Type: `Automation`
- Integration: `Gmail`
- Technology: `Schedule`
- Domain: `Email-Management`

**AI Chat Agent:**
- Status: `DRAFT`
- Type: `AI-Agent`
- Integration: `Postgres`, `Ollama`
- Technology: `AI`, `LLM`, `Multi-Agent`

**Tool/Utility:**
- Status: `Production`
- Type: `Tool`
- Integration: `Google-Drive`
- Domain: `File-Processing`

## Migration Plan

### Phase 1: Core Cleanup
1. Fix all typos in existing names
2. Apply status tags (Production, DRAFT, etc.)
3. Apply type tags (Tool, AI-Agent, etc.)

### Phase 2: Rename Standardization
1. Apply naming convention to all workflows
2. Use consistent category prefixes
3. Remove version numbers from names

### Phase 3: Tag Enrichment
1. Add integration tags
2. Add technology tags
3. Add domain tags

### Phase 4: Maintenance
1. Document tag usage in workflow descriptions
2. Create tag cleanup workflow (Meta category)
3. Regular quarterly reviews

## Tools for Implementation

### Manual Update Checklist
For each workflow:
- [ ] Check name against naming convention
- [ ] Fix any typos
- [ ] Add minimum required tags (Status + Type)
- [ ] Add integration tags
- [ ] Add technology tags if applicable
- [ ] Add domain tag if helpful
- [ ] Update workflow description to match new name

### Bulk Operations
Use n8n MCP tools:
- `n8n_list_workflows()` - Get all workflows
- `n8n_update_partial_workflow()` - Update names/tags
- AI-powered analysis for tag suggestions
